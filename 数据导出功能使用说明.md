# 数据导出功能使用说明

## 📋 功能概述

数据导出功能允许用户将系统中的各类数据导出为CSV、Excel或JSON格式，支持单个数据类型导出和批量导出多种数据类型。

## ✨ 主要特性

### 1. 支持的数据类型

- **日志数据**：系统日志、应用日志等
- **告警数据**：安全告警、系统告警等
- **安全事件**：统一安全事件数据
- **Windows安全日志**：Windows系统安全日志
- **系统性能指标**：CPU、内存、磁盘、网络等性能数据

### 2. 支持的导出格式

- **CSV**：逗号分隔值文件，适合用Excel打开，性能最好
- **Excel**：Excel工作簿文件（.xlsx），支持多工作表
- **JSON**：JSON格式文件，适合程序处理
- **ZIP**：压缩包（批量导出），包含多个CSV文件

### 3. 导出模式

- **单个导出**：导出单一数据类型，支持详细的过滤条件
- **批量导出**：一次性导出多种数据类型，生成Excel多工作表或ZIP压缩包

## 🚀 使用指南

### 访问入口

1. 登录系统后，在左侧菜单找到"数据导出"
2. 或直接访问：`http://localhost:8000/#/data-export`

### 单个数据类型导出

#### 步骤1：选择导出模式
- 点击"单个导出"按钮

#### 步骤2：配置导出参数
1. **数据类型**：选择要导出的数据类型
2. **导出格式**：选择CSV、Excel或JSON
3. **时间范围**：选择数据的时间范围（可选）
4. **过滤条件**：根据数据类型设置特定的过滤条件

#### 步骤3：设置过滤条件（可选）

不同数据类型支持不同的过滤条件：

**日志数据**：
- 日志级别：INFO、WARN、ERROR、DEBUG
- 关键词：搜索日志内容

**告警数据**：
- 告警级别：低、中、高、严重
- 告警类型：自定义告警类型
- 状态：待处理、处理中、已解决、误报

**安全事件**：
- 事件类型：自定义事件类型
- 严重程度：低、中、高、严重

**Windows安全日志**：
- 事件ID：Windows事件ID
- 用户名：目标用户名

**系统性能指标**：
- 指标类型：CPU、内存、磁盘、网络

#### 步骤4：开始导出
- 点击"开始导出"按钮
- 等待文件下载完成

### 批量导出

#### 步骤1：选择导出模式
- 点击"批量导出"按钮

#### 步骤2：配置导出参数
1. **选择数据类型**：勾选需要导出的数据类型（可多选）
2. **导出格式**：
   - Excel (多工作表)：所有数据在一个Excel文件中，每种数据类型一个工作表
   - ZIP (多个CSV文件)：每种数据类型生成一个CSV文件，打包成ZIP
3. **时间范围**：选择数据的时间范围（可选）

#### 步骤3：开始导出
- 点击"开始导出"按钮
- 等待文件下载完成

## 📊 导出示例

### CSV格式示例

```csv
ID,时间戳,日志级别,消息,来源,用户,IP地址
1,2026-04-14 10:30:00,INFO,"用户登录成功",web-app,admin,192.168.1.100
2,2026-04-14 10:31:00,WARN,"登录失败尝试",web-app,guest,192.168.1.101
```

### Excel格式特点

- 自动调整列宽
- 标题行加粗并带背景色
- 支持多工作表（批量导出）
- 可直接在Excel中打开和编辑

### JSON格式示例

```json
[
  {
    "id": 1,
    "timestamp": "2026-04-14T10:30:00",
    "level": "INFO",
    "message": "用户登录成功",
    "source": "web-app",
    "username": "admin",
    "ipAddress": "192.168.1.100"
  }
]
```

## 💡 使用建议

### 性能优化

1. **大量数据导出**
   - 建议使用CSV格式，性能最好
   - 合理设置时间范围，避免一次导出过多数据
   - 建议单次导出不超过10万条记录

2. **批量导出**
   - 如果数据量大，建议使用ZIP格式
   - Excel多工作表适合数据量较小的场景

### 数据筛选

1. **精确导出**
   - 使用过滤条件精确定位需要的数据
   - 合理设置时间范围
   - 使用关键词搜索

2. **定期导出**
   - 建议定期导出重要数据作为备份
   - 可以按天、周、月导出数据

### 文件管理

1. **文件命名**
   - 系统自动生成文件名，包含数据类型和时间戳
   - 格式：`{数据类型}_{时间戳}.{扩展名}`
   - 例如：`logs_2026-04-14-103000.csv`

2. **文件存储**
   - 导出的文件保存在浏览器默认下载目录
   - 建议定期整理和归档导出的文件

## 🔧 API接口说明

### 单个数据类型导出

```
GET /api/export/{dataType}?format={format}&startTime={startTime}&endTime={endTime}&...
```

**参数说明**：
- `dataType`: 数据类型（logs, alerts, events, security-logs, system-metrics）
- `format`: 导出格式（csv, excel, json）
- `startTime`: 开始时间（ISO 8601格式）
- `endTime`: 结束时间（ISO 8601格式）
- 其他过滤参数根据数据类型而定

### 批量导出

```
POST /api/export/batch?format={format}&startTime={startTime}&endTime={endTime}
Content-Type: application/json

["logs", "alerts", "events"]
```

**参数说明**：
- `format`: 导出格式（excel, zip）
- `startTime`: 开始时间（ISO 8601格式）
- `endTime`: 结束时间（ISO 8601格式）
- 请求体：数据类型数组

## ⚠️ 注意事项

1. **权限要求**
   - 需要登录系统才能使用导出功能
   - 某些数据类型可能需要特定权限

2. **数据量限制**
   - 单次导出建议不超过10万条记录
   - 如需导出更多数据，请分批导出

3. **时间范围**
   - 默认导出最近7天的数据
   - 可以自定义时间范围

4. **浏览器兼容性**
   - 支持Chrome、Firefox、Edge等现代浏览器
   - 建议使用最新版本浏览器

5. **文件大小**
   - CSV格式文件最小
   - Excel格式文件适中
   - JSON格式文件较大

## 🐛 常见问题

### Q1: 导出失败怎么办？
**A**: 
- 检查网络连接
- 确认时间范围内有数据
- 尝试减小时间范围
- 检查浏览器控制台错误信息

### Q2: 导出的文件打不开？
**A**:
- CSV文件：使用Excel或文本编辑器打开
- Excel文件：确保安装了Microsoft Excel或WPS
- JSON文件：使用文本编辑器或JSON查看器

### Q3: 导出速度慢怎么办？
**A**:
- 减小时间范围
- 使用CSV格式代替Excel
- 添加更多过滤条件
- 避免在系统高峰期导出

### Q4: 批量导出和单个导出有什么区别？
**A**:
- 单个导出：支持详细的过滤条件，适合精确导出
- 批量导出：一次导出多种数据，但过滤条件较少

### Q5: 导出的数据是实时的吗？
**A**:
- 是的，导出的是当前数据库中的最新数据
- 建议在数据稳定时导出
