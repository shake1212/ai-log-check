# 数据导出功能使用说明

## 功能概述

将系统数据导出为 CSV、Excel、JSON 或 ZIP 格式，支持单个导出和批量导出。

访问入口：`http://localhost:8000/#/data-export`

## 支持的数据类型

| 数据类型 | 表名 | 可用过滤条件 |
|---------|------|-------------|
| 日志数据 | log_entries | 日志级别、关键词、时间范围 |
| 告警数据 | alerts | 告警级别、类型、状态、时间范围 |
| 安全事件 | unified_security_events | 事件类型、严重程度、时间范围 |
| Windows安全日志 | windows_security_logs | 事件ID、用户名、时间范围 |
| 系统性能指标 | system_metrics | 指标类型、时间范围 |

## 导出格式

| 格式 | 扩展名 | 特点 |
|-----|--------|------|
| CSV | .csv | 文件小、性能好，适合大量数据 |
| Excel | .xlsx | 格式美观，支持多工作表 |
| JSON | .json | 结构化，适合程序处理 |
| ZIP | .zip | 批量打包多个CSV文件 |

## 使用方式

### 单个导出

1. 选择"单个导出"模式
2. 选择数据类型和导出格式
3. 设置时间范围和过滤条件（可选）
4. 点击"开始导出"

### 批量导出

1. 选择"批量导出"模式
2. 勾选需要导出的数据类型
3. 选择格式：Excel（多工作表）或 ZIP（多个CSV）
4. 点击"开始导出"

## API 接口

```
# 单个导出
GET /api/export/{dataType}?format={format}&startTime={startTime}&endTime={endTime}

# 批量导出
POST /api/export/batch?format={format}&startTime={startTime}&endTime={endTime}
Content-Type: application/json
["logs", "alerts", "events"]
```

## 注意事项

- 单次导出建议不超过 10 万条记录
- 默认导出最近 7 天数据
- 大数据量建议使用 CSV 格式
