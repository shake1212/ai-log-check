## 核心功能

### 用户视角

1. **前端UI配置**
   - 打开/关闭规则引擎
   - 调整超时时间
   - 实时生效

2. **自动化流程**
   - Python采集器自动读取配置
   - 根据配置启用/禁用规则引擎
   - 无需手动修改代码

3. **数据增强**
   - 自动添加威胁分数
   - 自动添加威胁等级
   - 自动添加匹配规则信息

### 技术视角

1. **配置管理**
   - 数据库存储配置
   - API提供配置读取
   - Python采集器动态加载

2. **规则引擎集成**
   - 模块化设计
   - 可选启用
   - 性能可控

3. **数据流程**
   ```
   前端UI → 数据库 → API → Python采集器 → 规则引擎 → 增强数据 → 后端存储
   ```

## 工作流程

### 配置流程
```
1. 用户在前端UI配置
   ↓
2. 前端调用API保存配置
   ↓
3. 配置保存到数据库
   ↓
4. Python采集器启动时读取配置
   ↓
5. 根据配置初始化规则引擎
```

### 运行流程
```
1. Python采集器收集事件
   ↓
2. 检查规则引擎是否启用
   ↓
3. [如果启用] 调用规则引擎分析
   ↓
4. 事件增强（添加威胁信息）
   ↓
5. 发送到Java后端存储
   ↓
6. 前端展示增强后的数据
```
