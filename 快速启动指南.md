# 规则引擎UI配置 - 快速启动指南

## 🚀 5分钟快速启动

### 前置条件
- 已安装 Java后端已安装
- 已安装 前端已安装
- 已安装 Python环境已配置
- 已安装 数据库已运行

### 步骤1: 数据库迁移（1分钟）

**选项A: 自动迁移（推荐）**
```bash
# 启动Java后端时会自动执行Flyway迁移
cd back-system
mvn spring-boot:run
```

**选项B: 手动迁移**
```sql
-- 连接到数据库，执行以下SQL
ALTER TABLE log_collector_configs 
ADD COLUMN IF NOT EXISTS enable_rule_engine BOOLEAN NOT NULL DEFAULT TRUE,
ADD COLUMN IF NOT EXISTS rule_engine_timeout INT DEFAULT 10;
```

### 步骤2: 启动服务（2分钟）

**终端1 - Java后端**
```bash
cd back-system
mvn spring-boot:run
# 等待启动完成，看到 "Started AiLogSystemApplication"
```

**终端2 - 前端**
```bash
cd ai-log-system
npm start
# 等待启动完成，自动打开浏览器
```

**终端3 - Python采集器**
```bash
cd back-system/src/scripts
python unified_log_collector.py
# 看到 "规则引擎状态: 启用" 表示成功
```

### 步骤3: 配置规则引擎（1分钟）

1. 打开浏览器访问 `http://localhost:8000/#/log-collector`

2. 在"采集器配置"表格中点击"配置"按钮

3. 找到新增的配置项：
   - 已安装 **启用规则引擎分析**: 打开开关
   - 已安装 **规则引擎超时（秒）**: 设置为 10

4. 点击"确定"保存

5. 重启Python采集器（Ctrl+C 然后重新运行）


## 🔧 常见问题

**Q: 采集器显示"规则引擎已禁用"？**
A: 在前端UI打开"启用规则引擎分析"开关，然后重启采集器

**Q: 采集器显示"规则引擎集成模块未找到"？**
A: 确保 `rule_engine_integration.py` 文件在 `back-system/src/scripts/` 目录

**Q: 规则引擎分析失败？**
A: 检查Java后端是否运行，访问 `http://localhost:8080/api/rule-engine/statistics` 测试

**Q: 如何禁用规则引擎？**
A: 在前端UI关闭"启用规则引擎分析"开关，然后重启采集器
