# 规则引擎UI配置 - 实施检查清单

## 实施前检查

在开始实施前，请确认以下条件：

- [ ] Java开发环境已安装（JDK 17+）
- [ ] Node.js环境已安装（Node 16+）
- [ ] Python环境已安装（Python 3.8+）
- [ ] 数据库已运行（MySQL/PostgreSQL）
- [ ] Maven已安装
- [ ] npm已安装
- [ ] 有数据库管理权限

## 实施步骤

### 步骤1: 数据库迁移

- [ ] 备份现有数据库
- [ ] 执行迁移脚本 `V1.0.5__add_rule_engine_config.sql`
- [ ] 验证新字段已添加：
  ```sql
  DESC log_collector_configs;
  -- 应该看到 enable_rule_engine 和 rule_engine_timeout 字段
  ```
- [ ] 检查现有配置是否已更新：
  ```sql
  SELECT id, name, enable_rule_engine, rule_engine_timeout 
  FROM log_collector_configs;
  ```

### 步骤2: Java后端更新

- [ ] 确认 `LogCollectorConfig.java` 已更新
- [ ] 编译Java项目：
  ```bash
  cd back-system
  mvn clean compile
  ```
- [ ] 检查编译是否成功（无错误）
- [ ] 启动Java后端：
  ```bash
  mvn spring-boot:run
  ```
- [ ] 验证后端启动成功（看到 "Started AiLogSystemApplication"）
- [ ] 测试配置API：
  ```bash
  curl http://localhost:8080/log-collector/configs/default
  ```
- [ ] 确认响应包含 `enableRuleEngine` 和 `ruleEngineTimeout` 字段

### 步骤3: 前端更新

- [ ] 确认 `LogCollectorService.ts` 已更新
- [ ] 确认 `index.tsx` 已更新
- [ ] 安装依赖（如有新增）：
  ```bash
  cd ai-log-system
  npm install
  ```
- [ ] 启动前端：
  ```bash
  npm start
  ```
- [ ] 验证前端启动成功（自动打开浏览器）
- [ ] 访问 `http://localhost:8000/#/log-collector`
- [ ] 点击"配置"按钮
- [ ] 确认配置模态框显示规则引擎选项：
  - [ ] "启用规则引擎分析"开关
  - [ ] "规则引擎超时（秒）"输入框
- [ ] 测试保存配置功能

### 步骤4: Python采集器更新

- [ ] 确认 `unified_log_collector.py` 已更新
- [ ] 确认 `rule_engine_integration.py` 文件存在
- [ ] 检查Python依赖：
  ```bash
  cd back-system/src/scripts
  pip install requests psutil
  ```
- [ ] 运行测试脚本：
  ```bash
  python test_rule_engine_integration.py
  ```
- [ ] 确认测试结果：
  - [ ] 模块导入测试通过
  - [ ] 客户端初始化测试通过
  - [ ] 事件分析测试通过
  - [ ] 采集器集成测试通过
  - [ ] 配置加载测试通过
- [ ] 启动Python采集器：
  ```bash
  python unified_log_collector.py
  ```
- [ ] 检查采集器日志：
  - [ ] 看到 "规则引擎集成模块已加载"
  - [ ] 看到 "规则引擎客户端已初始化"
  - [ ] 看到 "规则引擎状态: 启用"

### 步骤5: 端到端测试

- [ ] 所有服务都在运行：
  - [ ] Java后端（8080端口）
  - [ ] 前端（8000端口）
  - [ ] Python采集器
- [ ] 在前端UI配置规则引擎：
  - [ ] 打开 `http://localhost:8000/#/log-collector`
  - [ ] 点击"配置"按钮
  - [ ] 启用"规则引擎分析"
  - [ ] 设置超时时间为10秒
  - [ ] 保存配置
- [ ] 重启Python采集器
- [ ] 等待采集器收集事件（约5分钟）
- [ ] 检查采集器日志：
  - [ ] 看到 "开始规则引擎分析 X 个事件..."
  - [ ] 看到 "规则引擎分析完成: 分析=X, 匹配=Y, 失败=0"
- [ ] 查询数据库验证：
  ```sql
  SELECT 
      event_type,
      severity,
      threat_level,
      threat_score,
      matched_rule_count
  FROM unified_security_events
  WHERE threat_score IS NOT NULL
  ORDER BY timestamp DESC
  LIMIT 10;
  ```
- [ ] 确认事件包含威胁分析结果
