# 数据导出功能测试指南

## 🧪 测试准备

### 1. 环境要求
- Java后端已启动（端口8080）
- 前端已启动（端口8000）
- 数据库中有测试数据

### 2. 测试数据准备
确保数据库中有以下测试数据：
- 日志数据（log_entries表）
- 告警数据（alerts表）
- 安全事件（unified_security_events表）
- Windows安全日志（windows_security_logs表）
- 系统性能指标（system_metrics表）

## 📝 测试用例

### 测试用例1：单个数据类型导出（CSV格式）

**测试步骤**：
1. 访问 `http://localhost:8000/#/data-export`
2. 确保"单个导出"模式已选中
3. 选择数据类型：日志数据
4. 选择导出格式：CSV
5. 选择时间范围：最近7天
6. 点击"开始导出"

**预期结果**：
- 浏览器下载一个CSV文件
- 文件名格式：`logs_2024-04-14-HHMMSS.csv`
- 文件可以用Excel打开
- 数据格式正确，包含标题行

**验证点**：
- [ ] 文件成功下载
- [ ] 文件名正确
- [ ] 文件可以打开
- [ ] 数据完整
- [ ] 中文显示正常

```
Content-Type: application/json

["logs", "alerts", "events"]
```

## 📊 性能测试

### 测试指标

| 数据量 | 格式 | 预期时间 | 预期文件大小 |
|-------|------|---------|-------------|
| 1,000条 | CSV | <1秒 | <100KB |
| 1,000条 | Excel | <2秒 | <200KB |
| 10,000条 | CSV | <5秒 | <1MB |
| 10,000条 | Excel | <10秒 | <2MB |
| 100,000条 | CSV | <30秒 | <10MB |

### 性能测试步骤

1. **准备测试数据**
   ```sql
   -- 插入10万条测试日志
   INSERT INTO log_entries (timestamp, level, message, source, username, ip_address, created_at)
   SELECT 
     NOW() - INTERVAL FLOOR(RAND() * 30) DAY,
     ELT(FLOOR(1 + RAND() * 4), 'INFO', 'WARN', 'ERROR', 'DEBUG'),
     CONCAT('Test log message ', n),
     'test-source',
     'test-user',
     CONCAT('192.168.1.', FLOOR(1 + RAND() * 255)),
     NOW()
   FROM 
     (SELECT @row := @row + 1 AS n FROM 
       (SELECT 0 UNION ALL SELECT 1 UNION ALL SELECT 2 UNION ALL SELECT 3) t1,
       (SELECT 0 UNION ALL SELECT 1 UNION ALL SELECT 2 UNION ALL SELECT 3) t2,
       (SELECT 0 UNION ALL SELECT 1 UNION ALL SELECT 2 UNION ALL SELECT 3) t3,
       (SELECT 0 UNION ALL SELECT 1 UNION ALL SELECT 2 UNION ALL SELECT 3) t4,
       (SELECT 0 UNION ALL SELECT 1 UNION ALL SELECT 2 UNION ALL SELECT 3) t5,
       (SELECT @row := 0) r
     ) numbers
   LIMIT 100000;
   ```

2. **执行性能测试**
   - 导出1000条数据，记录时间
   - 导出10000条数据，记录时间
   - 导出100000条数据，记录时间

3. **分析结果**
   - 对比不同格式的性能
   - 检查内存使用情况
   - 验证文件大小
