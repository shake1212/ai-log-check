# AI评分优化技术文档

## 优化目标

- 完善程度：从70%提升到95%+
- 检测准确率提升30%，误报率降低40%

## 核心改进

### 1. 评分融合机制

```
最终得分 = 时序得分 × 0.3 + 关联得分 × 0.2 + 规则得分 × 0.4 + ML得分 × 0.1
```

动态权重规则：
- CRITICAL级别：权重 × 1.4
- HIGH级别：权重 × 1.2
- 关联分数 > 0.7：权重 × 1.2
- 权重归一化确保总和为1.0

### 2. 自适应阈值

```
动态阈值 = mean + k × std  (k=2.5)
```

- 滑动窗口：50条数据
- 最小数据量：10条（不足时使用固定倍数1.5）
- 每小时自动刷新阈值

### 3. 时序异常检测

- **Z-score检测**: >3.0 得分0.9, >2.0 得分0.7, >1.5 得分0.5
- **突增检测**: 当前值 > 近期均值×2 得分0.8
- **突降检测**: 当前值 < 近期均值×0.5 得分0.6

### 4. 关联分析

- **同一IP多异常**: 5分钟内 ≥3次 得分0.9, ≥2次 得分0.7
- **同一用户多异常**: 5分钟内 ≥3次 得分0.85, ≥2次 得分0.6

## 检测流程

```
规则匹配 → 频率/行为/网络/统计异常检测 → 时序异常检测 → 关联分析 → 加权融合 → 应用结果
```

结果应用：最终得分 > 0.6 标记为异常，> 0.9 为 CRITICAL，> 0.7 为 HIGH，其余为 MEDIUM。

```yaml
anomaly:
  detection:

    adaptive-window: 50
    zscore-k: 2.5
    min-data-count: 10
```
